System Development Guidelines

This document serves as the single source of truth for the Land Caller application logic, defining user tiers, GHL permission sets, Siftr integration requirements, and billing workflows.

1. Roles & CRM Permissions Matrix

Enterprise User

role_id: enterprise_client

Priority

Clients on 3, 6, or 12-month contracts. They require a dedicated sub-account with nearly full agency features unlocked.

CRM Cost: Included in Contract
Data Rate: Wholesale (Preferred)

Basic User

role_id: basic_client

Transactional clients purchasing one-time lead packages (Tier 1-3). Strict feature limitations to encourage upsells.

CRM Cost: Free w/ Package (Limited)
Data Access: Credit Based (Included)

Feature / Capability Basic Enterprise

Customize Pipelines (Stages/Names) Rename Only
Edit Call Scripts

3. Data & Siftr Logic

Integration Architecture

Siftr is white-labeled as "Land Caller Data" and embedded via iframe into the GHL sidebar. The behavior changes based on user Role.

Enterprise Logic
  • Limit: Unlimited Record Pulls.
  • Pricing Model: Wholesale (Discounted).
  • Payment Flow:
    1. Client initiates purchase in dashboard.
    2. Stripe Connect auto-splits transaction:
      • Data Provider Fee → Siftr
      • Platform Commission → Land Caller
  • Tiered Pricing Logic:
    User TypePrice/Record
    Public / Non-Contract $0.12
    Enterprise (Contract) $0.11 (Save $0.01)
  • NOTE: Data costs are NOT included in the monthly contract fee.
Basic Logic
  • Limit: Credit-based (Finite).
  • Logic:
    User buys Tier Package (LC receives $) -> Siftr API receives "Credit Grant" trigger -> User logs in -> Dashboard shows specific # of records to pull -> User pulls data -> Data delivered to LC for dialing.
  • Payment Flow:
    1. Client purchases Package (e.g. $1,100).
    2. Stripe Connect auto-splits transaction:
      • Fixed Data Cost → Siftr (Immediate)
      • Service Margin → Land Caller (Immediate)
  • Non-Dialing Users: Any user buying data without a campaign pays a higher per-record rate.

4. User Lifecycle States

1

Active Campaign / Contract

Client is currently paying for dialing services (Enterprise or Basic).

CRM_ACCESS = FULL (Enterprise) / LIMITED (Basic)
DATA_ACCESS = ENABLED
2

Post-Campaign / Maintenance Mode

Dialing has stopped. Contract ended or Basic package fulfilled. Client chooses not to renew dialing.

Requirement: Client must pay a monthly "Software Fee" to retain CRM access.
This fee is lower than the implicit software cost in Enterprise.
3

Cancelled / Inactive

Client cancels subscription or fails to pay Maintenance Fee.

Action: CRM Access is revoked (Shut Off).
Data Policy: Data is retained in GHL sub-account (frozen), but user cannot login. Allows for future reactivation.
